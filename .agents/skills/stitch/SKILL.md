# Google Stitch UI Design Skill

Google Stitch is an experimental AI design tool developed by Google Labs that allows you to generate UI designs and front-end code (HTML, CSS, JSX) from natural language prompts, images, or sketches. 

By connecting Google Stitch via the Model Context Protocol (MCP), you can enable AI agents to natively interact with your Stitch projects.

## Prerequisites
1. **Google Stitch MCP connected**: Ensure the Model Context Protocol for Google Stitch is properly configured in the AI Assistant's settings.

## Core Capabilities
- **Generate Screens**: Create new UI layouts based on user requirements.
- **Fetch Code**: Extract exact HTML, CSS, or React (JSX) code generated by the Stitch engine.
- **Design DNA**: Utilize consistent fonts, colors, and layout rules derived from an existing `DESIGN.md` file or an initial screen.

## Standard Workflow
1. **Understand Request**: Analyze the user's description of the component or screen (e.g., "Create a patient dashboard for a physiotherapy app").
2. **Design via Stitch**: Use the Stitch MCP tools to prompt the generation of the UI.
3. **Retrieve Artifacts**: Fetch the generated code and assets from the Stitch server.
4. **Integration**: Create or update local project files (`.tsx`, `.css`) with the fetched code.
5. **Refinement**: Make any necessary aesthetic or logical adjustments requested by the user, ensuring the "Design DNA" is respected.
